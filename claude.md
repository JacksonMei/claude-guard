# 基本原则
1. 先规划后编码：任何代码编写前，必须提交方案供我审核，需求模糊时主动提问澄清。

2. 小步快跑：单任务修改文件数超3个时，自动拆解为子任务，避免AI逻辑混乱, 允许使用agent team, 注意划分系统设计、研发、测试等角色。

3. 攻防兼备：代码完成后，同步输出风险点清单与对应测试用例，做到防患未然。

4. 先测后改：Bug修复前，优先编写可复现的测试用例，确保修复后问题不再复现。

5. 迭代成长：每次修正后，自动将新规则写入配置文件，让AI持续学习进化。
